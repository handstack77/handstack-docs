syn.$v: 폼 컨트롤의 필수 입력, 정규식 패턴, 범위, 사용자 정의 규칙에 대한 유효성 검사 기능을 제공합니다.

속성

유효성 검사 중 첫 실패에서 즉시 중단할지(false) 아니면 모든 규칙을 계속 검사할지(true) 여부입니다. 기본값은 true 입니다.
검증 실패 시 누적되는 오류 메시지 배열입니다. toMessages() 호출 전까지 값이 유지됩니다.
setElement()로 등록된 노드별 pattern/range/custom 검증 규칙을 담고 있는 맵입니다.
setElement()로 마지막에 설정된 대상 노드(엘리먼트)의 id 입니다.
numeric, email, juminNo, url, ipAddress, date, mobilePhone 등 pattern() 검증에서 사용할 수 있는 사전 정의된 정규식 모음입니다.
Root, Administrator, Master, Architect, Manager 등 권한 등급과 값을 매핑한 사전 정의된 객체입니다.
valueType은 HTML5 ValidityState와 유사한 값 검증 상태 코드, validType은 required/pattern/range/custom 검증 종류 코드를 정의합니다.

메서드

유효성 검사 정보를 수신할 노드를 대상(targetEL)으로 설정합니다.
사용자 입력을 수신할 노드에 필수 입력 검사 검증을 설정합니다.
사용자 입력을 수신할 노드에 정규식 패턴 검사 검증을 설정합니다.
사용자 입력을 수신할 노드에 숫자 입력 범위 검사 검증을 설정합니다.
사용자 입력을 수신할 노드에 사용자 기능 검사 검증을 설정합니다.
removeValidate(validType, validID)는 특정 검증 종류의 규칙 하나를, remove(validID)는 pattern/range/custom 전체에서 해당 validID 규칙을 제거합니다. (removeValidate 호출 전 setElement()로 대상을 지정하세요.)
등록된 모든 노드의 검증 규칙과 메시지, 대상 노드 정보를 초기화합니다.
입력한 id에 해당하는 단일 노드의 등록된 검증 규칙을 실행하고 결과(boolean)를 반환합니다.
지정한 노드 목록(txt_required, txt_pattern, txt_range, txt_custom)의 검사 검증을 실행합니다.
setElement()/pattern()/range()/custom()으로 등록된 모든 노드들의 검사 검증을 실행합니다.
누적된 오류 메시지를 줄바꿈으로 연결한 문자열로 반환하고, 내부 messages 배열은 비웁니다.
getRoleValue(roleNames)는 역할 이름을 roles 값으로, getRoleName(roleValues)는 반대로 값을 역할 이름으로 변환합니다.
