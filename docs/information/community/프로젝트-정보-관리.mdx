# 프로젝트 정보 관리

## HandStack 대화 주제

HandStack은 기업의 비즈니스 모델을 통합 하기 위해 검증된 개발 프로세스, 도구, 문서, 교육을 단순화하는 방안을 효과적으로 만드는 것을 연구 개발합니다. 이에 대해 대화 주제를 벗어나지 않도록 다음과 같은 기준을 정리합니다.

## 프로젝트(Project) 역할 분류
- (AL) ALL - 전체 멤버
- (CP) Customer Performer - 고객 업무 프로젝트 담당자
- (CC) Customer Consultant - 고객 업무 아키텍트, 컨설팅, 제안
- (CO) Customer Officer - 고객 업무/행정/관리 지원
- (CM) Customer Manager - 고객 업무 프로젝트 수행 관리
- (CD) Customer Developer - 고객 업무 프로젝트 화면/업무/데이터 중심 개발
- (CU) Customer UX/UI Designer - 고객 업무 프로젝트 UX/UI 중심 개발
- (BP) Business Performer - 도메인 업무 프로젝트 담당자
- (BC) Business Consultant - 도메인 업무 아키텍트, 컨설팅, 제안
- (BO) Business Officer - 도메인 업무/행정/관리 지원
- (BM) Business Manager - 도메인 업무 프로젝트 수행 관리
- (BD) Business Developer - 도메인 업무 프로젝트 화면/업무/데이터 중심 개발
- (BU) Business UX/UI Designer - 도메인 업무 프로젝트 UX/UI 중심 개발
- (PP) Project Performer - 수행사 프로젝트 담당자
- (PC) Project Consultant - 수행사 아키텍트, 컨설팅, 제안
- (PO) Project Officer - 수행사 업무/행정/관리 지원
- (PM) Project Manager - 수행사 프로젝트 수행 관리
- (PD) Project Developer - 수행사 프로젝트 화면/업무/데이터 중심 개발
- (PU) Project UX/UI Designer - 수행사 프로젝트 UX/UI 중심 개발

## 프로젝트 정보 관리
- 계정 
  - 가입
    - 가입 요청
    - 계정 확인
  - 관리
    - 가입 요청 확인
    - 회사 및 조직
    - 사용자 계정
- 프로젝트
  - 프로젝트 정보(생성|변경) 요청
  - 프로젝트 관리[조회/수정]
  - 사전업무
    - 업무 기획
    - RFP 요청서
    - 기획/제안서
  - 기술검토
    - 주관/참여사 관리
    - 시스템
      - 시스템 사양
      - 실행 환경
      - 개발 환경
      - 인프라 및 네트워크 환경
      - 검토 확인 (업무, 인프라, 보안등등 담당자 확인)
  - 업무합의 
    - 킥오프 미팅
    - 업무(주관부서/협력부서)담당 조직도
    - 업무(주관사/협력사)관련 조직도
    - 합의결과(기술/IT/비용등등 심의결과 고객 합의)
- 프로젝트 관리
  - 프로젝트 정보(생성|변경) 요청 접수
  - 프로젝트 관리
  - 프로젝트 담당자
  - 확장 시스템 설정
    - 메타
    - 배치 작업
    - 배포
    - 모니터링
    - 인프라
    - 프로젝트 관리
