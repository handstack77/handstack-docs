# prompter

`prompter`는 프롬프트 계약을 기반으로 LLM 요청을 실행하는 모듈입니다. OpenAI, Claude, Gemini, Ollama, LM Studio 같은 제공자를 설정으로 선택하고, `transact`에서는 `CommandType`을 `P`로 지정해 라우팅할 수 있습니다.

## 주요 기능

* XML 계약의 `<prompts>`와 `statement`를 읽어 프롬프트를 실행합니다.
* 계약별로 LLM 데이터소스를 선택합니다.
* KernelPlugin, MCP, CLI 도구 호출을 allowlist 기반으로 제한합니다.
* 코드도움 거래 결과를 프롬프트 본문에 삽입할 수 있습니다.
* 요청과 응답 로그를 모듈 로그 또는 `logger` 모듈로 남길 수 있습니다.

## 주요 설정

* `LLMSource`: Provider, 모델, endpoint, API Key를 정의합니다.
* `AllowedKernelPlugins`: 계약에서 사용할 수 있는 커널 플러그인과 함수 목록입니다.
* `AllowedMcpServers`: 실행 가능한 MCP 서버 allowlist입니다.
* `AllowedCliTools`: 실행 가능한 CLI 도구 allowlist입니다.
* `AllowedBodyFileBasePaths`: 프롬프트 body에서 읽을 수 있는 파일 경로 기준입니다.

## 주요 API

* GET: `/prompter/api/query/has`
* GET: `/prompter/api/query/refresh`
* GET: `/prompter/api/query/retrieve`
* GET: `/prompter/api/query/meta`
* GET: `/prompter/api/query/reports`
* POST: `/prompter/api/query/execute`

## 계약 위치

```txt
%HANDSTACK_HOME%/contracts/prompter/{ApplicationID}/{ProjectID}/{TransactionID}.xml
```

프롬프트 본문은 `${ParameterName}` 형식으로 요청 파라미터를 치환합니다. 인증, 헤더, body 속성은 `@ParameterName` 형식을 사용할 수 있습니다.

## transact 연동

```json
{
  "ServiceID": "GP01",
  "CommandType": "P",
  "ReturnType": "Json"
}
```

`RoutingCommandUri`에는 다음과 같은 규칙을 추가합니다.

```json
{
  "HDS|*|P|D": "http://localhost:8421/prompter/api/query"
}
```

## 운영 주의사항

LLM API Key와 외부 endpoint는 코드와 공개 문서에 직접 넣지 않습니다. 도구 호출은 계약 선언과 `module.json` allowlist가 모두 일치할 때만 허용되도록 관리해야 합니다.
