# 빠른 시작

> 전달 하고자 하는 요약 내용을 정리합니다

## 준비 중입니다
* 업무 담당자
* 화면 개발자
* 비즈니스 개발자
* 인프라 담당자
* 고객사 담당자

## 전달 하고자 하는 주요 내용은 다음과 같습니다
* 저) 직접 경험하고 실험한 과정이나 결과
* 술) 어떤 것을 분석하여 의미를 풀이하고 해석한 것
* 편) 산만하고 복잡한 자료를 편집해 질서를 부여한 것
* 집) 여러 사람의 견해나 흩어진 자료를 한데 모아 정리한 것

## 표(테이블) 만들기

마크다운을 이용해서 표를 그리는 방법은 다음과 같습니다.

### 기본 표

|제목|내용|설명|
|---|---|---|
|테스트1|테스트2|테스트3|
|테스트1|테스트2|테스트3|
|테스트1|테스트2|테스트3|
||||

### 텍스트 정렬
테이블 내의 텍스트는 *:* 문자로 정렬을 정의할 수 있습니다.

|제목|내용|설명|
|:---|---:|:---:|
|왼쪽정렬|오른쪽정렬|중앙정렬|
|왼쪽정렬|오른쪽정렬|중앙정렬|
|왼쪽정렬|오른쪽정렬|중앙정렬|


## 밑줄, 취소 선, 기울임꼴

밑줄을 사용하는 항목은 다음과 같습니다.
<u>HttpTriggerCSharp.cs</u>의 코드를 다음 코드로 바꿉니다.

기울임꼴을 사용하는 항목은 다음과 같습니다.
*HttpTriggerCSharp.cs*의 코드를 다음 코드로 바꿉니다.

취소 선을 사용하는 항목은 다음과 같습니다.
~~HttpTriggerCSharp.cs~~의 코드를 다음 코드로 바꿉니다.

## 마치며
스타일 지침은 엄격한 규칙이 아닙니다. 지침을 따르면 가독성이 손상되는 컨텍스트에서는 다른 방식으로 작업을 수행하세요. 예를 들어 대부분 코드 요소로 채워진 HTML 테이블에는 어디서나 코드 스타일이 아주 많이 적용될 수 있습니다. 해당 컨텍스트에서는 굵게 스타일 지정을 선택할 수 있습니다.
